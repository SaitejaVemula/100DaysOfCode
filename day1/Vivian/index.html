[Basics of Git]

Git Basic Concepts

There are centralized and distributed workflows. Centralized depends on a central server with everyone working off it. However, if the server is down, no one will be able to work or collaborate. Scalability is also more difficultas it depends on the server being able to handle capacity.

Distributed version control allows every team member has a copy of work and its history on a local machine. If the server is offline, work can still be synchronized with each other.

Git is a Version Control System (VCS) for distributed (decentralized) version control which allows multiple developers to coordinate work across the world.

Github, meanwhile, is an example of a popular repository.

+ Why Git? - Why Git?
  • Free
  • Open Source
  • Fast
  • Scalable
  • Cheap branching/merging options
  • Most compatable with current world standard use
+ What it does - What it does
  • Coordinates work between multiple developers
  • Keeps track of changes/history and when they were made by taking "snapshots" of work files
  • Allows you to take a snapshot by making a "Commit"
  • You can revisit any snapshot at any time
  • You can stage files before committing
  • Allows for programs to revert to a previous version in the history
  • Local and remote work options
  • Other workers can pull work onto their machines
  • Can create branches

+ Windows vs MAC/Linux - Windows vs MAC/Linux

\r carriage return

\n line feed

Windows: This is a sentence \r\n

Mac/Linux: This is a sentence \n

core.autocrlf configure this setting to prevent line and end issues between OS on a project (Not everyone will be using the same OS).

Windows: git config --global core.autocrlf true

Mac/Linux: git config --global core.autocrlf input

Basic GIT commands

  • git config --help This gives you the full menu.
  • git config -h This gives you summary of commands.
  • git init initializes local git repository - go into a folder via command line or terminal, this creates a .git folder withing a project
  • git add [file_name] adds/creates a file or files to the index in the staging area
  • git status check status of working tree (ie. check status within staging area to see what is ready for commit0
  • git commit commit changes in index
  • git push takes work from local and pushes to remote repository (such as github)
  • git pull pulls the latest version from remote repository (ie. making a copy to your local drive)
  • git clone clone/create a copy of repository into a new directory (ie. You want to create something based off of existing code, so you clone the original source and work on it in your own space)
